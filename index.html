Fundamentos de Programación en Java

¿Qué es Java y cómo funciona?

Java es un lenguaje de programación desarrollado por Sun Microsystems en 1995 y ahora es mantenido por Oracle. Es conocido por su versatilidad, seguridad y portabilidad.

JDK (Java Development Kit)

El JDK es un paquete de herramientas proporcionado por Oracle que contiene todo lo necesario para desarrollar aplicaciones en Java. Incluye el JRE (Entorno de Ejecución de Java), que a su vez contiene la JVM (Máquina Virtual de Java) y las bibliotecas de clases.

Componentes del JDK:

  1. JRE (Java Runtime Environment)
    • JVM (Java Virtual Machine): Ejecuta el bytecode (.class)
    • java.lang: Clases básicas como String, System
    • java.util: Colecciones, fechas
    • java.io: Manejo de archivos
    • java.net: Conexiones de red
  2. Herramientas de desarrollo
    • javac: Compilador (.java → .class)
    • java: Ejecutador de aplicaciones
    • javadoc: Genera documentación HTML
    • jdb: Depurador
    • jar: Crea archivos .jar (empaquetado)
  3. Bibliotecas adicionales
    • GUI: AWT/Swing, JavaFX
    • Bases de datos: JDBC
    • Seguridad: Criptografía
    • Multihilo: java.util.concurrent
  4. Herramientas de monitoreo/optimización
    • jconsole: Monitorea recursos (CPU, memoria)
    • jvisualvm: Perfilador de aplicaciones
    • javap: Desensambla bytecode
  5. Utilidades para desarrollo avanzado
    • native2ascii: Convierte caracteres Unicode
    • keytool: Manejo de certificados SSL

Fundamentos de programación

Los fundamentos de la programación consisten en el primer grupo de conceptos que tiene que entender un desarrollador. La lógica de programación siempre es la misma, lo que cambia es la sintaxis del lenguaje.

El Ciclo de Vida de un Programa

1. Entrada (Input)

Son los datos que tu programa necesita para trabajar. Puede ser texto escrito por un usuario, información de una base de datos o un clic del ratón.

2. Procesamiento (Processing)

Aquí ocurre la magia. Es el conjunto de instrucciones (el algoritmo) que toma la entrada, la manipula, hace cálculos y toma decisiones.

3. Salida (Output)

Es el resultado que el programa devuelve después del procesamiento. Puede ser un mensaje en pantalla, un archivo guardado o una acción realizada.

Tu Primer Contacto: "Hola, Mundo"

La tradición de todo programador es empezar con un programa "Hola, Mundo". Su objetivo no es ser complejo, sino verificar que tu entorno de desarrollo está bien configurado y entiendes la estructura básica para ejecutar código.

HolaMundo.java

                // Mi primer programa en Java
                public class HolaMundo {
                    public static void main(String[] args) {
                        // Esta línea imprime el texto en la consola.
                        System.out.println("¡Hola, Mundo desde Java!");
                    }
                }

¿Qué hace este código?

Este pequeño programa en Java muestra el mensaje "¡Hola, Mundo desde Java!" en la pantalla. Es el primer paso de muchos programadores porque ayuda a comprobar que todo está funcionando bien.

¿Cómo funciona? Imagina que el programa es como un mensajero: cuando lo ejecutas, va directo a la consola y deja el mensaje para ti. No necesitas entender palabras complicadas, solo saber que este código le dice a la computadora: "¡Escribe esto en la pantalla!".

Así, puedes ver que tu entorno de Java está listo y que puedes empezar a crear tus propios programas.

Tipos de Datos

En los fundamentos de programación tenemos tipos de datos primitivos junto a sus wrappers y los no primitivos, los tipos de datos son una categorización de los valores que puede almacenar una variable y las operaciones que se pueden hacer con ese tipo de dato.

Datos primitivos:

  • Los datos primitivos en Java son los que almacenan un valor en memoria, están almacenados en el Stack (memoria rápida para datos efímeros).
  • No pertenecen a un objeto.
  • No contienen métodos ni atributos.
  • Son medibles (por ejemplo: 0, 0.0, false).

Características Clave de los Tipos Primitivos

  • Almacenamiento en Stack: Ocupan memoria fija y directa, lo que los hace más rápidos que los objetos.
  • Inmutabilidad: Su valor no puede modificarse sin reasignación (ejemplo: int x = 5; x = 10;).
  • Eficiencia: Ideales para operaciones matemáticas intensivas por su bajo overhead.
  • Valores por Defecto: Si no se inicializan, Java les asigna un valor predeterminado (ejemplo: 0, 0.0, false).
  • Clases Wrapper: Cada tipo primitivo tiene una clase equivalente (ejemplo: int - Integer) para usos en colecciones o APIs que requieren objeto.

¿Por qué se convierten?

Se convierten principalmente para poder hacer operaciones de métodos y aplicarle lógica que lleva ese método con valores primitivos.

Variables y Palabras Reservadas

Una variable matemáticamente es la asignación de un carácter a un valor numérico perteneciente a un conjunto matemático. En programación una variable es un espacio en memoria el cual contiene un único dato (la cantidad de variables que puede almacenar una máquina depende de sus especificaciones de hardware como la RAM). Con ese dato asignado a la variable se pueden hacer múltiples operaciones según la lógica programada.

¿Qué es una palabra reservada?

Una palabra reservada o una keyword es un término especial definido por el lenguaje que tiene un significado específico y no puede ser usado como identificador (nombre de variable, método, clase, etc.).

Nota importante
Las palabras reservadas están en minúsculas y son case-sensitive.
Algunas palabras como true, false y null no son keywords, sino literales reservados, pero tampoco pueden usarse como identificadores.

Condicionales

En un entorno común una condición es un conjunto de acciones que se deben cumplir para que algo suceda.
Las plantas necesitan ciertas condiciones para crecer, como luz solar y agua. En Java, una condición es una expresión lógica (que evalúa a true o false) que determina el flujo de ejecución de un programa. A través de estructuras de control como if, else, switch o operadores ternarios, el programa decide qué bloque de código ejecutar basándose en si la condición se cumple (true) o no (false).

Estructura if-else

Tenemos la estructura if-else la cual sirve para determinar si una condición se cumple o no; por lo contrario se ejecutará la siguiente condición, el if-else es recomendado para cuando tenemos un programa ya con unas condiciones definidas algo que no crezca demasiado en el futuro para evitar unas condiciones en cascada, cuando tenemos N condiciones es mejor utilizar la expresión switch.

IfElseEjemplo.java
int edad = 18;
if (edad >= 18) {
    System.out.println("Eres mayor de edad.");
} else {
    System.out.println("Eres menor de edad.");
}

Estructura switch:

La estructura switch es un bloque de control que se utiliza para ejecutar uno o varios bloques de código basándose en el valor de una expresión por lo tanto es multidireccional.
Es especialmente útil cuando se necesitan comparar múltiples variables posibles con valor y ejecutar diferentes acciones según el caso.

Características Clave:

  1. Multidireccional:
    Evalúa una expresión una sola vez y dirige el flujo del programa a un bloque de código específico (case) según el valor coincidente.
  2. Tipos admitidos (desde Java 7+):
    • Tipos primitivos: byte, short, char, int.
    • Tipos envoltorio: Byte, Short, Character, Integer.
    • String (Java 7+).
    • enum.
  3. Componentes principales:
    • switch: Inicia la estructura.
    • case: Define un valor posible y su bloque de código asociado.
    • break: Termina la ejecución del switch (opcional, pero recomendado).
    • default: Bloque ejecutado si ningún case coincide (opcional).
Sintaxis general del switch
switch (expresion) {
    case valor1:
        // Código a ejecutar si expresion == valor1
        break;
    case valor2:
        // Código a ejecutar si expresion == valor2
        break;
    // ... más casos ...
    default:
        // Código si ningún caso coincide
}
Forma básica del switch
int dia = 3;
switch (dia) {
    case 1:
        System.out.println("Lunes");
        break;
    case 2:
        System.out.println("Martes");
        break;
    case 3:
        System.out.println("Miércoles");
        break;
    default:
        System.out.println("Otro día");
}
Switch con expresiones
int numero = 2 * 2;
switch (numero) {
    case 3 + 1:
        System.out.println("Cuatro");
        break;
    case 2 * 3:
        System.out.println("Seis");
        break;
    default:
        System.out.println("Otro número");
}

Operador ternario.

El operador ternario es una estructura condicional compacta que permite tomar decisiones en una sola línea. Es una versión simplificada de un if-else, diseñada para asignar valores o ejecutar expresiones basadas en una condición.

Componentes:

Condición: Expresión booleana que se evalúa como true o false.
(?) Separa la condición del valor si es verdadero.
(:) Separa el valor verdadero del falso.

Sintaxis del operador ternario
resultado = (condicion) ? valorSiTrue : valorSiFalse;

Bucles (Estructuras de Repetición)

Los bucles son bloques de control que permiten ejecutar un mismo código múltiples veces mientras se cumpla una condición. Son fundamentales para automatizar tareas repetitivas.

Estructura while (mientras)

Es una estructura de repetición que ejecuta un bloque de código mientras una condición es verdadera. La condición se evalúa antes de cada iteración, lo que significa que, si la condición es falsa desde el principio, el bucle no se ejecuta ni una sola vez.

Componentes:
  • Inicialización: Dejar la variable necesaria para poder ejecutar el código
  • Condición: Cuando la condición es verdadera se ejecuta el código
  • Avance: Siempre tiene que haber una o varias instrucciones que hagan que con cada iteración el ciclo esté más cerca de terminar
Concepto clave:
  • Iteración: Ejecutar repetidamente un conjunto de instrucciones dentro de un bucle.
WhileEjemplo.java
int contador = 1;
while (contador <= 5) {
    System.out.println("Contador: " + contador);
    contador++;
}

Salida:
Contador: 1
Contador: 2
Contador: 3
Contador: 4
Contador: 5

¿Cuándo Usar while?

  • Cuando el número de iteraciones es desconocido (ej: validaciones, lectura de datos).
  • Para condiciones complejas que no dependen de un contador simple.
  • En procesos interactivos (menús, juegos).
Consejo clave:

Siempre asegúrate de que la condición del while eventualmente sea false para evitar bucles infinitos. Usa break o actualiza las variables de control dentro del bucle.

Bucle do - while.

El bucle do-while es una estructura de control que ejecuta un bloque de código al menos una vez, y luego repite la ejecución mientras una condición sea verdadera. A diferencia del bucle while (que evalúa la condición antes de ejecutar el código), el do-while evalúa la condición después de la primera iteración.

Flujo de ejecución

  • Ejecuta el bloque de código dentro del do.
  • Evalúa la condición después de la ejecución.
  • Si es true, repite el proceso.
  • Si es false, sale del bucle.
DoWhileEjemplo.java
int contador = 1;
do {
    System.out.println("Contador: " + contador);
    contador++;
} while (contador <= 5);
// Salida:
// Contador: 1
// Contador: 2
// Contador: 3
// Contador: 4
// Contador: 5

Diferencias entre while y do-while

Característica while do-while
Evaluación de la condición Antes de ejecutar el bloque. Después de ejecutar el bloque.
Primera iteración Puede no ejecutarse (si la condición inicial es false). Siempre se ejecuta al menos una vez (incluso si la condición es false).
Uso típico Ideal cuando el número de iteraciones es desconocido y puede ser cero. Ideal cuando debes garantizar al menos una ejecución (ej: validaciones, menús interactivos).
Ejemplo común Validar entrada después de un primer intento. Mostrar un menú hasta que el usuario elija salir.

Bucle for.

El bucle for es una estructura de control determinista diseñada para ejecutar un bloque de código un número específico de veces, con un control preciso sobre:

  • Inicialización de una variable de control,
  • Condición de continuación,
  • Actualización del estado en cada iteración.

conceptos de la sintaxis

inicialización:
Es donde defines e inicializas la variable de control (generalmente un contador).
Se ejecuta solo una vez, antes de que el bucle comience. Ejemplo: int i = 0.

Condición:
Es una expresión booleana que se evalúa antes de cada iteración.
Si es true, el cuerpo del bucle se ejecuta; si es false, el bucle termina. Ejemplo: i < 5.

Actualización:
Es la operación que modifica la variable de control después de cada iteración.
Puede ser un incremento (i++), decremento (i--), u otra operación (i += 2). Ejemplo: i++.

ForEjemplo.java
for (int i = 0; i < 5; i++) {
    System.out.println("Valor de i: " + i);
}
// Salida:
// Valor de i: 0
// Valor de i: 1
// Valor de i: 2
// Valor de i: 3
// Valor de i: 4

Bucle for - each.

El bucle for-each (también llamado enhanced for loop) es una estructura simplificada diseñada específicamente para recorrer colecciones o arrays sin necesidad de manejar índices manualmente. Fue introducido en Java 5 para mejorar la legibilidad y reducir errores comunes.

ForEachEjemplo.java
int[] numeros = {1, 2, 3, 4, 5};
for (int num : numeros) {
    System.out.println("Número: " + num);
}
// Salida:
// Número: 1
// Número: 2
// Número: 3
// Número: 4
// Número: 5
Limitaciones del for-each
  • No permite modificar el array/lista original (solo funciona con copias).
  • No da acceso al índice (si necesitas el índice, usa for tradicional).
  • No es compatible con todas las estructuras de datos (ej: no funciona con Map directamente).

Comparación: for Tradicional vs for-each

Característica for Tradicional for-each
Requiere índice Sí (i = 0; i < length; i++) No
Modifica elementos Permite modificar el array/list. Solo lectura (a menos que uses objetos mutables).
Legibilidad Menos intuitivo Más claro y conciso
Uso Cualquier iteración con lógica compleja. Solo para recorrer colecciones

¿Por qué son importantes?

Control preciso: Permiten manejar casos excepcionales dentro de bucles.
Optimización: Evitan procesamiento innecesario (ej: con break).
Legibilidad: Hacen el código más claro al evitar anidamientos complejos.

consejos

Usa break y continue con moderación para no hacer el código difícil de seguir.
En su lugar, considera refactorizar con métodos pequeños.

Arrays (Arreglos)

Los Arrays son estructuras de datos que permiten almacenar múltiples valores del mismo tipo en una sola variable. Su tamaño es fijo; una vez definidos, no pueden crecer o reducirse.

¿Qué es un array?

Un array en Java es una estructura de datos que permite almacenar múltiples valores del mismo tipo en una sola variable. Los arrays tienen un tamaño fijo y cada elemento se accede mediante un índice (empezando desde 0).

Declaración y creación de un array
int[] numeros = new int[5]; // Crea un array de 5 enteros
Inicialización de un array
int[] numeros = {10, 20, 30, 40, 50};
Acceso y recorrido de un array
for (int i = 0; i < numeros.length; i++) {
    System.out.println("Elemento en posición " + i + ": " + numeros[i]);
}
// Salida:
// Elemento en posición 0: 10
// Elemento en posición 1: 20
// Elemento en posición 2: 30
// Elemento en posición 3: 40
// Elemento en posición 4: 50
Recorrido con for-each
for (int num : numeros) {
    System.out.println("Número: " + num);
}
// Salida:
// Número: 10
// Número: 20
// Número: 30
// Número: 40
// Número: 50

Casos de uso comunes

  • Almacenar listas de datos del mismo tipo (números, nombres, etc.).
  • Procesar grandes cantidades de información de manera eficiente.
  • Implementar estructuras de datos como pilas, colas o matrices.

Limitaciones de los arrays

  • Tamaño fijo: no se puede cambiar después de su creación.
  • No permiten almacenar diferentes tipos de datos en el mismo array.
  • Acceso manual por índice, lo que puede llevar a errores si no se controla bien.

¿Cuándo usar un array?

  • Cuando conoces de antemano la cantidad de elementos a almacenar.
  • Cuando necesitas acceso rápido y directo a los elementos por índice.
  • Cuando todos los elementos son del mismo tipo de dato.
